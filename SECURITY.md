# Security Policy

## Supported Versions

This project is currently in active development. Security updates and patches are provided only for the latest stable version.

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅ |
| < 1.0   | ❌ |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly and privately.

### Preferred Reporting Method

- Use **GitHub Security Advisories** to submit a private vulnerability report.

### Alternative Method

- Contact the maintainer directly via GitHub (Direct Message).

---

## What to Include

To help us resolve issues quickly, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Expected vs actual behavior
- Potential impact or severity
- Proof-of-concept, logs, or screenshots (if available)

---

## Response Timeline

- Initial response: within **3–5 business days**
- Status updates will be provided as investigation progresses.
- Confirmed vulnerabilities will be prioritized for fixing.

---

## Responsible Disclosure

Please do not disclose security vulnerabilities publicly until a fix has been released or coordinated disclosure has been agreed upon.

Public issues or discussions containing security vulnerabilities may be removed to protect users.

---

## Scope

This policy applies to:

- Application backend
- APIs
- Authentication and authorization
- Database access
- Infrastructure configuration related to the project

Third-party services or dependencies should be reported to their respective maintainers when applicable.

---

Thank you for helping keep this project secure.
